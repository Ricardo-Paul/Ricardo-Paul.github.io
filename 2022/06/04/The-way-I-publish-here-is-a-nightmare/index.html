The way I publish here is a nightmare

Ricardo Paul
On
June 4, 2022

I do not have a User Interface to write the blogs i publish here, when i first decided in 2021 that i needed a blog i only cared about getting the first post out and so for a couple of days i was head down on it.
I came across hexo which is the blogging framework powering this site, picked a theme and boom, the first piece “Why am I Starting a Blog?” was out. This all seems fast but there’s a catch, it requires some technical skills to set up and publish the notes since everything in here is written in markdown files inside the blog directory itself.

The entire content of the blog is also kept in a github repository. Instead of writing a post just as I would in say, a google doc with WYSIWYG support and pressing a publish button, I have to write a mix of html and markdown, tinkle with formatting and then run hexo specific CLI commands to publish. It doesn’t seem hard nor complex but it does add to the overhead of getting a blog post out. I shouldn’t have to have VSCode open when what I’m doing isn’t actual programming.

This needs to change though, the workflow to getting just plain text out the door should be very straightforward. I’ll need to allocate some time towards making my life easier in that sense, there’s also a few things i need to change about the theme too. I picked it from the collection available for hexo, i’m still loving the look but i’ll need to maybe clone the git repo and personalize it a bit more.
