Notes on processor architectures

Notes taken over the weekend about processor architectures

What does a 32-bit processor mean ?

One term to understand is “Address Space”.
Address space is the maximum size of memory (addressable memory) that a processor (mainly the addressing structure of a processor) can access.

At a high level, processors can access memory, but there’s a limit in how much memory they can access.

It’s important to note that one memory location is not 1 bit. But a collection of 8 bits. Or 1 byte.

An 8-bit processor can address 256 memory locations. That means 256 bytes, since each memory location is 1 byte (A unique 8-bit combination).
In C for example, the size of an int type is 4 bytes. That’s an int type variable fills 4 memory locations.

So the address space of an 8-bit processor is 256 bytes. That’s the maximum amount of memory it can access.
Similarly, a 32-bit processor can address 4 GB of memory. It’s 2^32 bytes, which is around 4GB.
Using the same formula, a 64-bit CPU (64 lines in the address bus ?) has an address space of 18 exabytes. (16 billion GP of RAM).

What directly determines how much memory a CPU can access is the size of the Address Bus. So an 8-bit computer will have an address bus that contains 8 lines. These 8 lines connect the CPU to the physical memory. Each of the lines can carry 1 bit.

According to this video, the CPU register size is what determines how many memory locations the CPU can access. A 2-bit register [0 | 1] can access 4 memory locations 2^2. That is because we can have 4 different combinations of 0 and 1.
The memory limit of the CPU comes from the register. My mental model now is that the CPU register should match the address bus.

CPU instruction set
An instruction set is a list of commands that the CPU can execute. Some example instructions can be LOAD (load something from RAM), ADD (add a number to) etc.. What we should retain here is that these instructions are coded in bits. If they are coded in say, 4 bits, the instruction set is called a 4-bit instruction set.
0000: ADD
0001: LOAD
Etc…
CPUs on the Market - Some history

CPUs with 16-bit registers in the past.
Intel 8086
Motorola 68000

Although their instructions were both coded in 16 bits, the instructions weren’t the same. So software written for one could not be executed by the other.

CPUs with 32-bit registers in the past
Intel 80386 (included the 16-bit instructions of the previous plus more)
Motorola PowerPC

Pentium from Intel (IA32)
IA32 means Intel Architecture 32 (32 bits CPU)
And then there was the Pentium CPU from Intel, which also had a 32-bit register.
Something to pay attention to when it comes to naming: a 16-bit intel processor was called Intel 8086, a follow-up 32-bit was called Intel 80386
86 is common in the names, hence the name x86.
So x86 refers to the Intel 32-bit architecture.

The pentium being a 32-bit CPU (has a 32-bit register) can only address 4 GB of memory. AMD is going to improve it.

Itanium (IA64) from Intel
For servers and high-end workstations, Intel created the Itanium which had a 64-bit register. It had an entire new instruction set, not compatible with any previous model. The Itanium architecture is called IA-64 or Intel Architecture 64.

Windows OS compatible with Intel’s Itanium - Windows XP
This is just for a better understanding. Windows XP was designed to be compatible with Intel’s IA-64. This was the first 64-bit CPU made by Intel - however, the standard for 64-bit CPU today isn’t the Itanium, as we’re going to see.

Athlon 64 from AMD
AMD is another CPU maker. They’ve been making really good CPUs. They made the Athlon 64 which was an improvement of Intel’s Pentium.
Need to underline that AMD CPUs are like functional clones of Intel’s. One of their selling points was you can buy an AMD CPU and still run all your Intels software.

So AMD made the Athlon 64 which supported Intel’s pentium instruction set but added some features that allow it to address more than 4GB of memory.
AMD’s Athlon 64 became so popular that Microsoft began to design their operating systems to the Athlon64 standards. And the standard was called the 64-bit standard or AMD64.
Even Intel’s future versions of their Intel’s CPU (started with the Pentium 4 CPU) were adapted to AMD’s 64-bit standard.
Remember x86 for the 32-bit Intel’s CPU ?
Intel’s CPU adapted to the 64-bit standard are called x64
So x64 CPUs are more advanced than x86 since they are based on 64-bit architecture, while x86 are based on 32-bit architecture.

Windows Operating systems that are x86-64 means they support both Intel’s old 32-bit and the new 64-bit standard. The AMD improvement.
